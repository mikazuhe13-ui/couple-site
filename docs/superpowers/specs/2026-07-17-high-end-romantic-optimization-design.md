# 高级浪漫情侣纪念网站优化设计

## 目标

在保留网站真实内容与明亮浪漫基调的前提下，优先解决首屏媒体加载、Hero 排版失衡、相册交互性能、移动端体验和搜索分享信息不完整的问题。最终页面应当像一本有空间层次的私人纪念册，而不是常见的婚礼模板。

正式站点为 `https://couple-site-dusky.vercel.app`，公开名称使用“杜明洋 × 陈柯嘉”。

## 视觉方向

### 色彩

- 瓷白底色：`#FFFBF5`
- 玫瑰粉：`#D4898C`
- 香槟金：`#C9A96E`
- 象牙纸面：`#FFF3E8`
- 深可可正文：`#3D2B2B`
- 柔和次级文字：`#6B5A52`

所有组件从同一组 CSS 变量取色，移除组件内含义重复但数值不同的粉色与金色。

### 字体

- 展示标题：Playfair Display
- 中文正文与标题：Noto Serif SC
- 小型界面标签：系统无衬线字体

字体通过 `next/font/google` 自托管并生成稳定的字体变量，避免外部字体样式表阻塞渲染和字体切换造成布局偏移。

### 视觉签名

网站的唯一强视觉记忆点是“会呼吸的香槟金记忆长廊”：照片位于有纵深的空间舞台中，随着鼠标位置产生克制、有规律的整体视差；单张照片按自身空间坐标产生差异化位移和倾斜。其他区域以纸张、留白、细线和柔和淡入为主，避免特效竞争。

## 页面与组件设计

### Hero

Hero 保留全屏视频和真实姓名，但取消 `vh` 负边距拼接布局。内容使用稳定的三段式网格：姓名位于视觉中心偏上，副标题紧随其后形成一个排版组，入口按钮位于下方但仍属于同一阅读节奏。桌面端与移动端分别使用 `clamp()` 控制字号、间距和最大行宽。

视频使用已有桌面文件，并增加静态 poster。页面不再请求不存在的移动端视频。视频默认 `muted`、`playsInline`、`loop`，采用 `preload="metadata"`；首屏可见且用户未开启节省流量或减少动态效果时才播放，页面不可见时暂停。移动端播放失败时稳定显示 poster，不注册全局滚动重试监听。

桌面端视频层使用基于 Hero 自身滚动进度的轻量纵向视差。减少动态效果、触摸设备和节省数据模式关闭视频自动播放与视差。

### 空间相册

桌面端相册由普通 CSS columns 改为可控的 3D 舞台：

- 舞台使用 `perspective`，卡片分为前、中、后三个深度层。
- 鼠标位置映射为舞台的有限位移与旋转，变化通过 Framer Motion 的 MotionValue 和 spring 完成，不进入 React state。
- 每张卡片基于固定空间坐标获得不同的 `x`、`y`、`z` 和初始旋转，整体移动时保持规律而非随机抖动。
- 卡片悬停最多向前移动约 `36px`、缩放约 `1.035`，离开后柔和回位。
- 进入视口时按空间层次落位，只动画 `transform` 和 `opacity`。

相框采用四层结构：暗香槟金外沿、浅金斜切面、象牙白内衬、细微内阴影。方向性高光跟随指针但透明度受控，避免塑料质感。照片说明位于相框下沿的纸质铭牌区域，保证可读性。

移动端使用可横向拖动、带惯性和吸附的空间轨道。当前卡片居中并轻微前浮，邻近卡片露出边缘提示可继续滑动。移动端不依赖 hover 或持续跟手倾斜。

灯箱支持上一张、下一张、键盘方向键、Esc 关闭、焦点管理、背景滚动锁定和明确的无障碍名称。手机端支持左右滑动切换。

所有照片使用 `next/image`，提供准确 `sizes`、固定宽高比和模糊占位。仓库内照片转换为 WebP；Supabase 远程图片通过受限的 `remotePatterns` 加载，不开放任意远程域名。

### 章节过渡与其他动效

各 section 共用一套 reveal 变体与缓动曲线：

- 标准进入：轻微上移、透明度变化。
- 重点标题：细线展开后文字进入。
- 图片内容：缩放与透明度组合，不使用大面积实时 blur。

滚动视差只用于 Hero 和空间相册。动画遵守 `prefers-reduced-motion`；触摸设备减少持续动画。所有可持续循环的装饰动画在离开视口时停止。

### 导航与移动端

导航拆为独立组件。桌面端维持精简章节导航；手机端菜单按钮达到至少 `44 × 44px`，包含 `aria-expanded`、`aria-controls` 和动态“打开/关闭菜单”名称。

手机菜单打开时锁定背景滚动，支持 Esc 关闭，展示明确关闭图标，并将焦点移入菜单。点击链接后关闭并恢复焦点。正文最小字号与行高保证中文可读性，section 留白按手机视口统一收紧。

## 性能与架构

首页保持服务端页面壳，将需要交互的部分拆成客户端岛：

- Hero 媒体控制与视差
- 导航与移动菜单
- 空间相册及灯箱
- 音乐控制
- 留言表单

不需要实时交互的章节尽可能保留为服务端或纯展示组件。非首屏重型交互通过 `next/dynamic` 延后加载。Framer Motion 改用 `LazyMotion` 与 `m`，只加载所需动画功能。

Supabase 内容在服务端获取并传入初始数据；留言提交保留客户端行为。若服务端数据读取失败，继续使用现有 fallback 数据，不阻断页面渲染。倒计时作为独立客户端组件每秒更新，避免带动整页重渲染。

图片尺寸稳定、字体自托管、Hero poster 和明确的容器比例共同控制 CLS。首屏只优先加载 poster 或必要媒体，不优先加载相册图片。目标是减少首页初始 JS、改善 LCP、CLS 和 INP。

## SEO 与分享

根布局补全：

- 包含真实姓名的 title、description、keywords 和 author
- `metadataBase`、canonical
- Open Graph 网站类型、站点名、标题、描述、URL、中文区域和分享图
- Twitter Card
- 搜索引擎 robots 指令

新增 `app/sitemap.js` 和 `app/robots.js`。首页输出 JSON-LD，使用 `WebSite` 与 `Person`/关系描述，避免虚构婚姻状态或其他未提供事实。管理页标记为 `noindex`，不进入 sitemap。

分享图使用专门的静态或 Next.js 生成图片，不从视频帧动态截取，确保社交平台稳定抓取。

## 代码质量与可访问性

- 拆分过大的 `CoupleSite` 与 `components/ui/index.jsx`，让导航、音乐、留言和 section header 各自拥有清晰边界。
- 静态导航配置移出组件，事件回调用 `useCallback` 仅在确有下游收益时使用。
- 不在鼠标移动时调用 `setState`。
- 交互元素补充可见焦点、可访问名称、语义 heading 层级和键盘路径。
- 图片 alt 使用真实说明；纯装饰元素标记为隐藏。
- 表单包含显式 label、提交状态和错误反馈。
- 对 API 返回做基础状态检查，避免静默吞掉所有失败。

## 错误处理

- 视频不可播放：显示 poster，不重复打扰用户。
- Supabase 内容失败：显示 fallback 内容；留言失败时保留输入并显示可恢复错误。
- 图片失败：保留稳定相框尺寸并显示柔和占位。
- 灯箱图片索引随数据变化失效时自动关闭。

## 验证

每个实现阶段独立提交并至少执行与改动相关的检查。合并前执行：

- `npm run lint`
- `npm run build`
- 桌面端与移动端浏览器检查
- 键盘导航、Esc、焦点与 reduced-motion 检查
- 不存在资源请求、图片尺寸和控制台错误检查
- 首页 metadata、JSON-LD、sitemap 和 robots 输出检查
- Lighthouse 或等价的 Core Web Vitals 基线/改后对比
